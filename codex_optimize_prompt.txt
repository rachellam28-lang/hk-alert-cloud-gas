## 重點優化方向

### 1. Performance — 2,777 stocks scraping 太慢
- 目前每個 shard serial scrape ~463 stocks × ~11s = ~85 min per shard
- `scraper.py` 入面 scrape_stock 係 single-threaded
- 建議：每個 shard 內部用 ThreadPoolExecutor（參考 runner.py L264 _scrape_parallel）
- 小心 rate limiting — HKEX 會 block concurrent requests

### 2. Robustness — failed stocks retry
- 目前 failed_stocks 只記錄唔 retry
- `scraper.py` abort threshold 7/12（12 個 request 入面 >=7 fail 就 abort）
- 成個 shard abort 會 skip 好多 stock
- 建議：failed stocks 喺 shard 最後 retry 一次，或者 merge 階段補 scrap

### 3. Data Completeness
- ccass.json 應該有晒 2,777 隻正股，唔係 500
- verify `_export_json` 冇隱藏 LIMIT
- verify `get_active_stocks()` return 2,777（而家係 ✓）

### 4. Frontend Optimization
- ccass.html ~1,300 lines, 88KB — 可唔可以 split JS/CSS？
- 2,777 stocks 嘅 DOM rendering performance
- 搜尋 + filter + sort 速度
- Mobile responsive check

### 5. DB Maintenance
